Data Types in JavaScript

There are two types of data types in JavaScript.

  1. Primitive (Value Types)

  2. Non Primitive (Reference Types)

We can classify primitive data types in 7 categories.

  1. number
  2. string
  3. boolean
  4. undefined
  5. null
  6. bigint
  7. symbol

There is only one type of non-primitive data type that is called object.

In JavaScript, objects are collections of key-value pairs. The keys are strings (or Symbols) and the values can be any type of data, including other objects.

Objects are further divided into three categories:

  • Object Literals

    An object literal is a comma-separated list of key-value pairs wrapped in curly braces. They are the simplest way to create objects. For example: 

    
const person = {
  name: "John",
  age: 30,
  city: "New York"
};

  • Arrays

    Arrays are special types of objects used to store ordered collections of values. Each value in an array is associated with a numeric index. For example: 

    
const numbers = [1, 2, 3, 4, 5];

  • Functions

    Functions are first-class objects in JavaScript, meaning they can be stored in variables, passed as arguments to other functions, and returned from functions. For example: 

    
function greet(name) {
  return `Hello, ${name}!`;
}

Object.freeze() and Object.seal()

  • Object.freeze()

    The Object.freeze() method freezes an object. A frozen object can no longer be changed; freezing an object prevents new properties from being added to it, existing properties from being removed, and existing properties, or their enumerability, configurability, or writability, from being changed. The object becomes immutable.

    For example:

    
const obj = {
  prop: 42
};

Object.freeze(obj);

obj.prop = 33; // This will not work
console.log(obj.prop); // 42

  • Object.seal()

    The Object.seal() method seals an object. Sealing an object prevents new properties from being added to it and marks all existing properties as non-configurable. Values of present properties can still be changed as long as they are writable.

    For example:

    
const obj = {
  prop: 42
};

Object.seal(obj);

obj.prop = 33; // This will work
console.log(obj.prop); // 33

delete obj.prop; // This will not work
console.log(obj.prop); // 33

Both Object.freeze() and Object.seal() are used to control the mutability of objects, but Object.freeze() makes an object completely immutable, while Object.seal() allows changes to existing properties but prevents the addition or removal of properties.
