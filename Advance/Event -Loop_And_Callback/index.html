Event Loop and Callback Queue

Event Loop

The Event Loop is a fundamental part of JavaScript's concurrency model. It enables JavaScript to perform non-blocking operations, even though JavaScript itself is single-threaded. The event loop continuously checks the call stack and the callback queue and executes the functions from the callback queue once the call stack is empty.

How It Works:

  1. JavaScript executes the code in the call stack.
  2. If there are asynchronous operations (e.g., setTimeout, fetch), their callbacks are placed in the callback queue.
  3. When the call stack is empty, the event loop moves callbacks from the callback queue to the call stack and executes them.

Callback Queue

The Callback Queue (also known as the task queue) holds the functions that are waiting to be executed after the current code has finished executing. Callbacks are added to the callback queue when an asynchronous operation completes.

Types of Queues:

  • Macro Task Queue: Includes tasks like setTimeout, setInterval, and I/O operations.
  • Micro Task Queue: Includes tasks like Promise callbacks and MutationObserver callbacks. Micro tasks are processed before the macro tasks.
