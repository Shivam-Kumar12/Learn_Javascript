About Hoisting

Hoisting is a JavaScript mechanism where variables and function declarations are moved to the top of their containing scope during the compile phase. This means that you can use functions and variables before they are declared.

Variable Hoisting

Variables declared with var are hoisted to the top of their function scope. However, the assignment is not hoisted.

    
// Example of Variable Hoisting
console.log(hoistedVar); // Output: undefined
var hoistedVar = 'This variable is hoisted';
console.log(hoistedVar); // Output: This variable is hoisted

Function Hoisting

Function declarations are hoisted to the top of their containing scope, meaning you can call a function before it is declared.

    
// Example of Function Hoisting
hoistedFunction(); // Output: This function is hoisted

function hoistedFunction() {
  console.log('This function is hoisted');
}
